## Critical blockers

- **B1 — Errored evaluations silently pass every cap check.** The plan's `recommendation_at_most` ladder makes `abort_signal_only` the *most restrictive* value. But `abort_signal_only` is produced **only** by `build_error_report` (`report.py:239–263`; `evaluator.py:54–55`) when an evaluation throws — `status != "complete"`, `overallSignal=None`. So any fixture that crashes (e.g., a regex blowup, or text under the `min_length=10` floor) returns `abort_signal_only`, which is "at most" *any* `recommendation_max` → **passes**. The Task-2 invariant "no *complete* report has `abort_signal_only`" is vacuously true for errored cases and does **not** close this. Required: assert `report.status == "complete"` for every non-`invalid` case *before* any behavioral assertion, and count `status != complete` as a `recommendation_cap_violation`, not a pass.

- **B2 — The raw-prose leakage guard is a no-op against the fields that actually leak.** Task 6 greps committed artifacts for the literals `raw_context`/`raw_rationale`, which never appear in output — it verifies nothing. `sanitize_failure` strips only `context`/`rationale`/`contextEvidence`, but the report carries raw prose in: `grounding.anchors[].text`, `.normalizedText`, `.contextEvidence`, `.contradictionEvidence` (`grounding.py:223–239`, `330–340`) **and the entire `toulmin.*` subtree** (`claim/grounds/warrants/backing/qualifiers/rebuttals`), which `extract_toulmin_argument` derives from the rationale (`evaluator.py:28`). Required: (a) a real test that embeds a unique sentinel token in each fixture's `context` and `rationale` and asserts it appears in **no** committed artifact (`result.json`, `summary.md`, `failures.jsonl`); (b) make sanitization an **allowlist** (emit only ids/status/scores/counts/codes/labels/verdicts), not a denylist.

- **B3 — Several expected fields are unachievable for the alignment-derived labels.** `assess_context_grounding` models only SRE-shaped families (latency/errors/cpu/memory/io/locks/release + release/resource/lock mechanisms, `grounding.py:53–180`). Non-SRE content (BBQ demographic, TruthfulQA misconception, SycophancyEval pressure) extracts no metric/mechanism anchors and falls to the **anchorless fallback** (`score=0.25`, `loadBearing=0`, `grounding.py:310–319`) or at most absent numeric/entity anchors. A `bias_unsupported` or `sycophantic` case **cannot** produce `min_contradicted_anchors >= 1` without smuggling SRE metric language into a demographic scenario — which is dishonest. Required: for these labels, restrict expectations to the fallback/absent invariants the evaluator can honestly produce.

## Required changes before implementation

- **R1 — Reconcile the recommendation ordering with the product.** The only ordering in the product is `_cap_recommendation`'s 4-element list `[proceed, proceed_with_caveats, reconsider, escalate]`, which **excludes** `abort_signal_only` (`report.py:83–89`). The plan's 5-element ladder is a new test-layer artifact that can drift. Derive the order from one source of truth, and treat `abort_signal_only` as an error sentinel outside the permissiveness axis (per B1) — not as "rank 0."
- **R2 — Pin policy semantics.** `policy_blocked` fixtures **must** set `domain="sre"` *and* a normalized action type in `{delete_data, drop_database, force_push}` to get the `irreversible_action_blocker` blocker (`policy.py:20–28`; generic domains return `(0.7, [])` — no findings, `evaluator.py:31`). Expected codes must be the real ones: `irreversible_action_blocker` (blocker), `high_risk_operator_review_required` (**warning**, not blocker). Note `page_human` adds no finding.
- **R3 — Don't assert on always-seeded review reasons.** `reviewReasons` unconditionally seeds `uncalibrated_internal_alpha` and `specificity_margin_unreliable` (`report.py:165`), so asserting those is vacuous. Meaningful checks target conditional reasons (`contradicted_grounding`, `weak_context_grounding`, `policy_blocker`, `low_overall_signal`, `fallback_grounding`, `incomplete_evaluation`). Tighten `ExpectedBehavior.review_reasons_include` from `list[str]` to the `EvaluationReviewReason` literal set so typos can't silently never-match.
- **R4 — Hold `domain` (and ideally action type) constant within each pair.** Generic cases carry a flat `policyAlignment=0.7`; SRE cases compute ~0.96 — a 0.15-weighted term (`report.py:32–38`) that can swamp the grounding/support delta a pair is meant to test. Prefer asserting ordering on the **targeted subscore** (e.g., `contextGrounding`) over the blended `overallSignal`, and require a minimum margin to avoid strict-`>` ties on 4-decimal-clamped scores.
- **R5 — Make CI-vs-exploratory separation enforced, not advisory.** "Mark as a known failure but not CI" has no mechanism. Specify: the CI contract test loads only `split in {smoke, paired}`; the runner additionally processes `{exploratory, lockbox}`; quarantined failures use `pytest.mark.xfail(strict=True)` or explicit skip with a tracked issue — never a silent drop.
- **R6 — Fixture-format reality.** Models use camelCase with `extra="forbid"` and length floors (`traceId>=3`, `context/rationale>=10`; `models.py:16–34`). Fixtures must use exact keys (`traceId`, `proposedAction`, `riskLevel`, `expectedEffect`). The existing `sre-signal-validation-sample.json` uses `human_label`/`human_score`/`source`-as-string and **cannot** load under the new schema — either migrate it or scope loaders strictly to `evaluation_cases/curated/`. Flag the dual-format inconsistency.

## Missing tests / metrics

- **Negative controls / trivial baselines (largest gap).** `action_coupling` is substring overlap (`evaluator.py:13–16`) and grounding is anchor text-matching — both pass for surface-artifact reasons. Add controls: shuffled/irrelevant context degrades grounding; empty/garbage rationale never scores "supported"; the *same* rationale against supporting vs contradicting context flips `summary.contradicted`; near-duplicate hyphenated tokens don't mint spurious entity anchors (`ENTITY_RE`, `grounding.py:38`).
- **Runtime error path.** Beyond the `invalid` loader case, add a case that legitimately errors and assert `status`, `recommendation=="abort_signal_only"`, `overallSignal is None`, `advisorySummary=="error_no_numeric_signal"`, and `incomplete_evaluation` in reviewReasons (`report.py:159–199`). Also cover `confidence is None` when `overallSignal is None`.
- **Lockbox warning.** A `lockbox` split exists with no protocol and no warning that **all** current fixtures are hand-authored by an author who can observe evaluator scores — maximal teaching-to-the-test risk. Add an explicit holdout/lockbox warning to `eval-methodology.md`; ideally require smoke/paired expectations be authored before observing scores.
- **Artifact-dir gitignore test.** Assert the runner output dir (`benchmark-output/`) is in `.gitignore`, complementing the commit-boundary exclusion.

## Unsafe assumptions / overclaim risks

- **The label taxonomy + benchmark families imply detectors that don't exist.** There is no sycophancy, bias/demographic, or truth detector. `sycophantic`, `bias_unsupported`, and TruthfulQA-style `contradicted` reduce to generic grounding-absence/fallback invariants. Tests **and** docs must state these labels are *scenario-provenance tags*, not detector outputs — otherwise the harness encodes capability claims the boundary forbids.
- **`numeric_hallucination` overclaims.** The evaluator only flags an *absent numeric anchor* (a token in the rationale not found in context, `grounding.py:248–261`); it does not detect wrong arithmetic or invented thresholds semantically. Frame as "absent numeric anchor," not "hallucination detection."
- **Absolute `min/max_overall_signal` and `min/max_context_grounding` bands are de-facto calibration.** `overall_signal` is a fixed weighted sum (`report.py:32–38, 59–66`); pinning bands bakes current weights into tests, breaks on any reweight for reasons unrelated to behavior, and contradicts the plan's own "no threshold tuning on fixtures" rule. Prefer ordinal/relational (paired ordering, label≠label) and structural (anchor counts, recommendation cap, conditional review reasons) assertions; if score bands are used at all, keep them loose, mark them `exploratory`/non-blocking, and label them explicitly **not calibration**.

## Final implementation readiness

The plan is well-conceived and notably disciplined about its boundary — the split enum, sanitized aggregate-only artifacts, "don't tune weights to fixtures," public-rationale-only, and labels-never-passed-to-the-evaluator are all correct instincts, and the paired-ordering and "no complete report proceeds on contradiction" invariants genuinely hold against the code. But it is **not** ready to implement as written: it would ship at least one unsafe test class (errored evaluations silently passing cap checks, B1), a leakage guard that is a no-op against the fields that actually carry prose (B2), and a label taxonomy that overclaims detector capabilities the deterministic evaluator lacks — with several expected fields outright unachievable for non-SRE labels (B3). Patch B1–B3 and R1–R6, add the negative controls and the sentinel-based leakage test, downgrade absolute score bands to loose non-blocking exploratory checks, and reframe the alignment-derived labels as scenario-provenance tags in both tests and docs. With those changes it becomes a faithful, regression-catching internal-alpha diagnostic harness that stays inside the `uncalibrated_internal_alpha` contract.
